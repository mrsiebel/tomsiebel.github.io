基于hexo+github搭建的个人博客：Siebel

         一直有这个想法，总想去写点什么，文章？技术要点？常识？历史政治？军事？…..不管什么也好，总是要写要记要存的，但网络时代，手机时代让我们失去了这种能力，甚至到了提笔忘字，不愿意安心读完一篇文章，一本书，感觉这是我们这一辈的悲哀.

         写点东西总感觉不简单，怀疑自己文笔不好怎么办，被骂怎么办… 后来一想可能你都可能没有读者，哪来的那些有的没的，但难道就不写了么，你又不是为了别人去写东西，想通了就好啦

      然后又有一个问题摆在面前，写什么呢？？在这里当然写技术啦，但自己技术感觉也就是个小白.写个人感情？是不有点矫情啦，让别人窥见你的内心有什么好.写历史政治？虽然了解，可也不深刻…..后来想想什么烂七八糟的，不如想起啥写啥，技术点看到了就记下来，没事看看……

         写到这里我就想啦，随便写,那我们的心目中的博客是什么呢？看到了别人写的这个我就豁然开朗啦

         “我所以指的写博客，不单只是写一篇文章出来这一结果。而应该是写的这一过程，写过技术文章的朋友应该跟我一样有这么一个过程。

  • 自己了解学习，文章所涉及到的知识点，及知识点衍生出来的知识点。

  • 对学习的知识点进行验证，以确保理论值与实践值保持一致

  • 构思文章的大纲，哪些部分需要重点写，需要配合实例代码，图片等信息

  • 动手写，写完后再次检查校正并排版，然后发表

  • 针对网友的评论中提出的问题进行回复”
